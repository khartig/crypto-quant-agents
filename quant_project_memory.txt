Manual remount (non-persistent) after reboot/login
1) Ensure drive is mounted by Ubuntu at /media/kevin/QUANTDATA.
2) Run:
   sudo mkdir -p /mnt/quant-data
   if ! mountpoint -q /mnt/quant-data; then
     sudo mount --bind /media/kevin/QUANTDATA /mnt/quant-data
   fi

Driver/workaround note
- UAS quirk configured for Seagate bridge 0bc2:2344:
  /etc/modprobe.d/seagate-uas-quirk.conf
  options usb-storage quirks=0bc2:2344:u

Boot behavior note

Quick start checklist (daily/operator use)
1) Verify external storage is mounted to /mnt/quant-data
   - findmnt /mnt/quant-data
2) Activate virtual environment
   - source .venv/bin/activate
3) Verify Ollama model availability
   - ollama list
   - confirm llama3.1:8b is present
4) Ingest fresh market data (example)
   - quant-phase1 ingest --exchange kraken --symbol BTC/USDT --timeframe 1h --limit 300
5) Run agent plane with stable timeout (example)
   - OLLAMA_TIMEOUT_SECONDS=180 quant-phase1 agent-plane --exchange kraken --symbol BTC/USDT --timeframe 1h --strategy-model llama3.1:8b --ops-model llama3.1:8b
6) Generate visual diagnostics
   - quant-phase1 visualize-run
7) Review outputs in the latest run directory
   - strategy_proposal_signal.json
   - backtest_evaluation.json
   - risk_decision.json
   - paper_trade_intent.json
   - visuals/*.png
