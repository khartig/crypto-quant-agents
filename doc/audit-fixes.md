# Audit Fixes Implementation Plan
## Problem statement
Recent repo-specific audits found that buy/sell profitability is overstated in research artifacts and too fragile after trading frictions. The current pipeline mixes cost-blind backtest metrics, static confidence/risk gates, and long-only execution semantics, which can produce high actionability but weak net profitability.
## Current state
Backtest and ensemble returns are computed without explicit fee/slippage deductions in `src/quant_agents/backtest.py (225-333, 418-485)`, while the risk gate consumes those outputs in `src/quant_agents/agent_plane.py (1980-2264, 2291-2606)`. Walk-forward validation similarly computes gross returns in `_run_walkforward_evaluation` (`src/quant_agents/agent_plane.py (947-1130)`). Trigger labels and model selection are threshold/accuracy centric in `src/quant_agents/trigger_model.py (184-228, 393-491)`. Paper execution charges fees but realized sell PnL currently excludes fee impact in `src/quant_agents/paper_trading.py (191-226)`. Ensemble online updates currently increment observations when an arm is selected, even if no execution occurred, in `src/quant_agents/agent_plane.py (522-594)`.
## Proposed changes
### 1) Add a shared cost model and net metrics for backtest and walk-forward
Introduce explicit transaction-cost inputs (fee bps + slippage bps) and turnover-derived cost deductions in backtest/walk-forward computations. Update `run_sma_backtest`, `run_ensemble_backtest`, and `_run_walkforward_evaluation` so each artifact reports gross and net return streams, turnover units, total cost drag, and break-even bps diagnostics. Keep existing gross metrics for comparability but make net metrics first-class in downstream gating.
### 2) Make risk gating and calibration net-aware
Extend risk observations and thresholds to include net-return and cost-drag evidence (from updated backtest/walk-forward outputs). Keep deterministic policy structure in `risk_runner`, but prioritize net profitability signals when deciding pass/block/fail. Add explicit reason codes for cost-drag rejections so blocked runs indicate friction-related causes instead of only confidence-level failures.
### 3) Improve trigger-model actionability objective and reporting
Retain the current Gaussian-NB foundation for this pass but shift training/evaluation selection to a net-expectancy-oriented objective: compute per-threshold actionable expected value after estimated cost and persist selected-threshold diagnostics in model artifacts. Expand training metrics to include actionable precision/recall, per-class precision/recall/F1, and calibration-oriented confidence diagnostics alongside existing accuracy/confusion outputs.
### 4) Fix paper-execution realism and codify sell semantics
Update paper execution so realized PnL deltas are net of transaction costs on exits, and add configurable execution slippage (same sign conventions used in backtests). Keep the system long-only for now (no synthetic shorting), but make sell semantics explicit as long-reduction/close-only behavior in code-level reasoning and docs to prevent long/short interpretation drift.
### 5) Tighten adaptive-ensemble learning signals
Refine `_update_ensemble_weight_state` and `_compute_arm_weights` so adaptive learning rewards executed, net-attributed outcomes and includes turnover/cost sensitivity. Avoid increasing arm observation counts for non-executed selections; track execution participation separately so exploration and quality signals do not overfit to blocked/unfilled decisions.
### 6) Thread configuration and CLI surface through all entrypoints
Add new settings/env vars and CLI options for backtest/walk-forward cost controls and execution slippage in `src/quant_agents/config.py`, `src/quant_agents/cli.py`, and `src/quant_agents/openclaw_native.py`, ensuring defaults preserve backward compatibility while enabling cost-aware operation without code edits.
### 7) Documentation and validation
Document all audit-fix behaviors, new config knobs, and metric interpretation in `doc/audit-fixes.md`, `.env.example`, and relevant README sections. Validate via `python3 -m compileall src scripts`, `scripts/validate_walkforward_policy_suite.py`, `scripts/verify_phase2_completion.py`, and targeted CLI smoke runs for `backtest`, `agent-plane`, and `train-trigger-model` to confirm contract compatibility and deterministic behavior.
