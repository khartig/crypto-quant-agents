# Trigger labeling and objective specification
## Purpose
This document defines the labeling/objective behavior used by the trigger model benchmark for roadmap item 9:
- richer outcome labels beyond purely directional framing,
- explicit trade/no-trade quality demotion,
- precision/recall/coverage frontier analysis under transaction costs.
## Labeling modes
Two labeling modes are supported by `src/quant_agents/trigger_model.py`:
- `directional_v1`
  - Labels are based on horizon return threshold checks.
  - Positive/negative actionability is primarily directional.
- `triple_barrier_v2`
  - Labels are built with an upper barrier, lower barrier, and time barrier.
  - Captures path-aware outcomes and allows clearer separation of ambiguous horizons.
## Trade-quality and meta-label objective
The model computes a per-sample quality score and applies:
- `trade_quality_pass` boolean gate (`trade_quality_min_score` threshold),
- demotion of low-quality actionable labels to `hold`,
- `meta_label` fields to distinguish actionable-quality opportunities from weak/uncertain cases.
This objective emphasizes **actionable signal quality** over gross directional hit-rate.
## Optimization target and frontier metrics
Training and threshold selection optimize execution-aware utility (post-cost), and emit:
- actionable precision/recall,
- actionable coverage (`actionable_rate`),
- expectancy metrics (`net_expectancy_per_bar`, `net_expectancy_per_actionable`),
- execution backtest metrics (fill/rejection/equity/PnL deltas).
Action-confidence threshold sweeps are stored under:
- `threshold_optimization.action_confidence_frontier.rows`
within each model artifact.
## Benchmark procedure (roadmap item 9 deliverable)
Run:
`python3 scripts/run_labeling_objective_benchmark.py --scenario-set default`
Default scenario set compares:
- directional without quality gate,
- directional with baseline quality gate,
- triple-barrier with baseline quality gate,
- triple-barrier with stricter quality gate.
Generated artifacts:
- JSON benchmark contract: `trigger_labeling_objective_benchmark.v1`,
- markdown comparison summary,
- actionable coverage vs precision frontier PNG plot.
